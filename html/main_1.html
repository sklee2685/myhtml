1)컴퓨팅적 사고와 문제 해결

1-1) 컴퓨팅적 사고와 문제 해결
* 컴퓨팅적 사고(Computational Thinking): 컴퓨터(사람이나 기계)가 효과적으로 수행할 수 있도록 문제를 정의하고,
 그에 대한 답을 기술하는 것이 포함된 사고 과정 일체를 일컫는다.


* 컴퓨터가 효과적으로 수행할 수 있도록 해주는 방법



1. 자료수집
2. 자료분석(자료의 이해, 패턴 찾기)
3. 구조화로 문제를 그래프,차트같은 시각화
4. 추상화(필요한 요소를 파악후 단순화함)
5. 자동화(컴퓨팅 시스템이 수행할 수 있는 형태로 해결책을 나타냄)
6. 일반화로 문제해결과정을 다른 문제에도 적용

4) 애플리케이션 구성

4-1) 기본 구성
안드로이드 스튜디오에서는 애플리케이션을 만드는 2가지 과정이 있다
첫 번째로는 XML을 이용하여 인터페이스 화면을 디자인(ex전자 출결앱 메인화면)을 하고,
자바 코드를 사용한다.


* 기본 개념


1. 액티비티(Activity)는 안드로이드에서 애플리 케이션을 구성하는 4가지(남은 3가지는 2-3 참고)의 컴포넌트 중 하나이다.(예를 들면 전화거는 화면을 만들 수 있음)
2. @Override: 메소드가 부모 클래스의 메소드를 재정의 한 것
3. onCreate()메소드: 액티비티가 생성되는 순간 한번만 호출하는 메소드로 모든 초기화 및 인터페이스 설정은 여기에 들어감
4. setContentView()라는 메소드는 사용자가 꾸민 액티비티의 화면을 설정하는 메소드이다.


4-2) 비주얼 도구 및 설명



  

    
    옆에 사진은 비주얼에서 주로 사용하는 도구들이 있는 창이다, 이곳에서 필요한 것을 드래그해서 인터페이스를 디자인 할 수 있다.

6-3) Listener 사용

사용방법


1. 리스너 클래스를 내부 클래스로 정의
2. 리스너 클래스를 무명 클래스로 정의( 가장 많이 사용함)
3. 리스너 인터페이스를 액티비티 클래스에 구현


*무명 클래스 정의


  

    setContentView(R.layout.activity_main);
Button button=(Button) findViewById(R.id.button);

MyListenerclass buttonListener=new
MyListenerclass();
button.setOnClickListener(buttonListener);

    단, 이 코드를 사용하기 위해선
오른쪽 코드처럼
 MyListenerclass라는 클래스를 만들어야함
 
    class MyListenerclass implements View.OnClickListener {
  public void onClick(View v) {
    Toast.makeText(getApplicationContext(), "버튼이 눌렸습니다.",
    Toast.LENGTH_SHORT).show();
  }
};

8) 캔버스

8-1) Canvas클래스 와 Paint 클래스
Canvas클래스: 그림을 그리는 캔버스(도화지)
Paint 클래스: 캔버스에 그릴 선의 스타일, 폰트, 도형 등 그리기 속성을 가지고 있는 클래스

*전체적인 구조


  

    
    * 첫 번째 상자에 그림을 그리는 코드를 넣는다. 
ex) canvas.drawLine()을 사용해 선을 만들 수 있다.

* 두 번째 상자에는 MyView를 생성하고 이것을 Activity의 콘텐트 뷰로 설정함
  




*색상
캔버스에서 색상으로는 주로 paint.setColor(Color.RED); 이런 식으로 색상 설정이 가능하다.
하지만 이렇게 마지막에 RED를 넣는 대신에 색의 3원색인 RGB(Red, Green, Blue)성분을 8비트로 표시된 코드를 넣어서도 사용이 가능하다.
ex) paint.setColor(Color.BLUE); == paint.setColor(Color.0x0000FF);
이런 식으로 파란색을 설정할 수 있다.
